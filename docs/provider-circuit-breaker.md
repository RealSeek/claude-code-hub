# Provider Circuit Breaker

供应商级熔断器面向多实例、高并发代理流量。状态和滚动窗口事件由 Redis Lua 脚本原子更新，进程内状态只作为快速读取和 Redis 不可用时的降级方案。

## 策略

- 统计窗口为 60 秒，至少累计 20 个请求后才按失败率判断。
- 失败率达到 40% 时打开熔断器。
- 连续失败快速通道默认从旧配置的 5 次提升为 8 次，用于真正的上游故障；`failureThreshold=0` 仍表示禁用。
- 第一次打开 60 秒，之后指数退避，最终不超过现有 `openDuration` 配置。
- 半开最多允许 2 个带租约的探测请求，成功达到现有 `halfOpenSuccessThreshold` 后关闭。
- 租约由 Redis 原子删除或自动过期；客户端错误、上下文超限、Key 级错误只释放租约，不计入供应商健康度。

## Redis 键

每个供应商使用状态 Hash、成功事件 ZSET、失败事件 ZSET 和半开租约 ZSET。事件只保留窗口及状态 TTL 内的数据。所有状态变更脚本使用 Redis 服务端时间，避免多实例本地时钟影响窗口边界。

## 兼容性

现有供应商配置字段和管理接口路径保持不变。新增窗口统计字段仅作为健康状态的附加信息；旧 Redis Hash 缺少这些字段时按零值读取。Endpoint 和 vendor/type 熔断器仍保持独立语义。
